# Исправление ошибки доступа к Docker

Если вы видите ошибку `Permission denied` при работе с Docker, выполните следующие шаги:

## Решение

### 1. Добавьте пользователя в группу docker:

```bash
sudo usermod -aG docker $USER
```

### 2. Примените изменения группы:

**Вариант A:** Выйдите и войдите заново в систему

**Вариант B:** Или выполните команду (без перелогина):
```bash
newgrp docker
```

### 3. Проверьте, что всё работает:

```bash
docker ps
```

Если команда выполняется без ошибок, всё готово!

## Альтернативное решение (временное)

Если не хотите добавлять пользователя в группу, используйте `sudo`:

```bash
sudo docker-compose up -d --build
sudo docker-compose logs -f
sudo docker-compose down
```

## Проверка текущих групп

Чтобы проверить, в каких группах вы состоите:
```bash
groups
```

Группа `docker` должна быть в списке.
